# Неделя 7 — Локальный RAG + сравнение local↔cloud

**Задание ментора:** подключить локальную LLM к RAG-пайплайну; индекс из недели 6;
retrieval — локально, генерация ответа — локальной моделью; сравнить ответы
локальной и облачной модели; оценить **качество**, **скорость**, **стабильность**.
**Результат:** ✅ RAG, работающий полностью локально (retrieval + генерация), плюс
воспроизводимый бенчмарк local↔cloud (`rag compare`) по трём осям.

Окружение: Apple M4 Pro / 24 GB · Ollama 0.31.1 · генерация `qwen2.5:7b` ·
эмбеддинги `nomic-embed-text` (768-dim) · индекс `kb` из недели 6 · облачный
базлайн `google/gemini-2.5-flash` (OpenRouter).

## 1. Что уже было и что добавлено

RAG-пайплайн уже был готов архитектурно: индекс `kb` из недели 6 построен на
**локальных** эмбеддингах (`ollama / nomic-embed-text`), а генерация идёт через
общий `LLMGateway`. После недели 6 (движок Ollama + тумблер провайдера) этого
достаточно, чтобы **и retrieval, и генерация шли локально** — см. раздел 3.

Для части «сравнить и оценить» добавлено (аддитивно, без изменения retrieval/генерации):

- **`jarvis/rag/compare.py`** — харнесс сравнения: прогоняет контрольные вопросы
  через локальную и облачную модель и считает качество / скорость / стабильность.
- **CLI `rag compare`** — команда в REPL.
- **Честность сравнения:** retrieval одинаков для обеих сторон (один индекс,
  локальные эмбеддинги; `rag_rewrite` принудительно выключен, чтобы запрос — а
  значит и найденные чанки — не отличались). Мета-оценку «ответ подтверждён
  источниками» делает **один фиксированный судья** (облачная модель) для обеих
  сторон, так что ни одна модель не оценивает сама себя.
- Тесты `tests/test_rag_compare.py` (герметичные).

## 2. Как это работает (пути кода)

- **Retrieval (локально):** запрос эмбеддится провайдером, записанным в самом
  индексе (`ollama/nomic-embed-text`), поэтому поиск локален независимо от
  провайдера чата.
- **Генерация (локально при тумблере):** `grounded_answer` → `answer` →
  `self._gateway` → `RoutingEngine` → локальная модель, когда `provider=ollama`.
- **Сравнение:** харнесс переключает `config.set("provider", …)` между прогонами
  (снимок/восстановление конфига), судья — на отдельном закреплённом gateway.

## 3. Запуск полностью локального RAG (без облака)

```bash
brew services start ollama          # локальный демон
ollama pull qwen2.5:7b              # генерация
ollama pull nomic-embed-text       # эмбеддинги (уже есть с недели 6)

cd ~/PycharmProjects/jarvis-cli && source .venv/bin/activate
unset OPENROUTER_API_KEY            # доказательство: облако недоступно
export JARVIS_LLM_PROVIDER=ollama
python -m jarvis
```
В REPL:
```
> config set rag on
> config set rag_index kb
> rag ask what is the invariant checker
```
Ответ строится из локально найденных чанков локальной моделью, с цитатами
(Sources/Quotes). Работает и при выключенном Wi-Fi.

## 6. Оценка

**Качество.** Retrieval общий и локальный → hit-rate 100% у обеих сторон (это
свойство индекса, не модели). По генерации облако немного впереди: покрытие
ожидаемых фактов 89% против 85%, «подтверждено источниками» 78% против 70%.
Цитирование нужного источника — 100% у обеих. Вывод: локальная 7B-модель на
внутренних вопросах по документации **сопоставима** с облаком, отставание
небольшое и в основном на «пограничных» формулировках.

**Скорость.** Облако быстрее в ~3.5 раза (2.3 с против 7.9 с в среднем; включает
локальный retrieval и, для локали, возможную подгрузку модели в память). Для
интерактивного чата это заметно; для фоновых/пакетных задач локальная скорость
приемлема.

**Стабильность.** Разброс задержек близок (CV 0.28 против 0.32). Разброс качества
внутри вопроса у локали чуть выше (0.05 против 0.00) — облако при temperature по
умолчанию отвечало почти идентично между повторами. «Флипы вердикта судьи» высоки
у обеих (60%/50%), но это артефакт малого числа повторов (2): один флип = не
единогласно; при `repeats=3+` метрика сглаживается. Важно: у локали **0% ошибок**,
у облака — **10%** (один транзиентный сбой API) — локальный вариант оказался
надёжнее по доступности (нет сети — нет отказов).

**Итог.** Полностью локальный RAG жизнеспособен: приватно, бесплатно, оффлайн,
надёжно по доступности; цена — скорость и небольшой недобор качества. Облако имеет
смысл там, где критична латентность или максимум качества.
